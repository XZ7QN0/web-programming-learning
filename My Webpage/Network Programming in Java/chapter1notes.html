Chapter 1: Basic Network Concepts

Network - A collection of computers and other devices that can send data to and receive data from one another in almost real-time.

  • Often connected by wires (bits of data turned into electromagnetic waves)
  • Wireless networks transmit data using radio waves

Nodes - Any device that can connect to a network

Host - A node that is a general-purpose computer

Layers in Detail

Host-to-Network: The parts of networking that is hidden from the developer, which defines how a particular network interface (Ethernet cord, WiFi antenna, or any hardware used for networking) sends IP datagrams over its physical connection to the local network and the world. Can also be known as:

  • Link layer
  • Data link layer
  • Network interface layer

Host-to-Network typically concerns a developer when the concern is about performance.

Internet Layer: Known as the network layer in the OSI model, this layer defines how:

  1. bits and bytes of data are organized into larger groups called packets, and
  2. the addressing scheme by which different machines find one another.

The most wifely used network layer protocol (and the only one that Java understands) is the Internet Protocl (IP), which is actually two protocols:

IPv4

  • 32-bit addresses
  • Accounts for >90% of Internet traffic
  • Little over 4 billion possible IP addresses
  • Datagram header is 20-60 bytes long
  • Payload that contains up to 65,515 bytes of data

IPv6

  • 128-bit addresses & additional routing features
  • Catching on fast
  • Datagram contains larger header
  • Up to 4GB of data
  • Will be enough IP address for everyone
  • Written in 8 blocks of 4 hexadecimal digits separated by colons

Transport Layer: The layer responsible for ensuring that packets are received in the order they were sent and that no data is lost or corrupted. If packets get lost or corrupted, the transport layer asks the sender to retransmit the packet. IP networks implement this by adding an additional header to each datagram that contains more information. There are two primary protocols at this level:

    TCP (Reliable Protocol)
  • High-overhead protocol that allows for retransmission of lost or corrupted data and delivery of bytes in the order they were sent

    • UDP (Unreliable Protocol)
  • Allows the receiver to detect corrupted packets but does not guarantee that packets are delivered in the correct order (or at all)
  • Is often faster than TCP

Application Layer: The layer that delivers data to the user. The three lower layers (Transport layer, Internet layer, and Host-to-Network layer) work together to define how data is transferred from one computer to another. It decides what to do with the data after it's transferred. Typically, data will be presented to a user in a form that they understand, rather than a long stream of numbers. The application layer is where most of the network parts of our programs will spend their time.

Some application layer protocols include:

  • HTTP (Web)
  • SMTP, POP, and IMAP (Email)
  • FTP, FSP, and TFTP (File Transfer)
  • NFS (File access)
  • Gnutella & BitTorrent (File sharing)
  • SIP & Skype (Voice communication)
  • & more

Protocols in Detail

    IP
  • Developed with military sponsorship during Cold War
  • Robust
  • Designed to allow multiple routes between any two points and to route packets of data around damaged routers
  • Open & platform-independent
  • Packets may not arrive in the order they were sent, or even arrive at all
    TCP
  • Layered on top of IP to give each end of connection the ability to acknowledge receipt of IP packets and request retransmission of lost or corrupted packets
  • Allows packets to be reorganized on receiving end in the same order packets were sent
  • Carries a good amount of overhead
    UDP
  • Chosen over TCP if order of data sent does not matter, and loss of individual packets won't corrupt data stream
  • Acceptable for applications where some data loss is not noticed by end user (ex: losing a few bits from video or audio signal)
  • Error-correcting codes can be built into UDP data streams at the application level for any missing data
    ICMP
  • Another protocol that can run on top of IP
  • Uses raw IP datagrams to relay error messages between hosts
  • Used in ping
  • Java doesn't support ICMP, nor allow sending of raw IP datagrams (Only supports TCP and UDP protocols, and application layer protocols built on top of these)
    The following layers and protocols can only be implemented in Java programs by linking to native code:
  • Transport layer
  • Internet layer
  • Lower layer protocols, such as
    • ICMP
    • IGMP
    • ARP
    • RARP
    • RSVP
    • Others
